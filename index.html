Caption Quest

A short top-down puzzle game made in 24 hours as a solo dev using PyGame for the University of Bristol 2022 GameJam.

The theme was 'It's not a bug, it's a feature!', and my 'bug' was the player character not having their movement restricted when talking to NPCs. This, for example, lets them walk over textboxes, grab keys out of them, and start multiple conversations at once. It was awarded the prize for 'Most Innovative Game' out of around 30 teams of 1-6 people.

Carpetman Listener

Building off of Caption Quest, this is a longer puzzle game following the same concept, made in PyGame and submitted to the Mental Health Game Dev Champions 2024 Jam on itch.io.

I designed 24 levels with some new features, such as textboxes with non-walkable borders, beam-activated switches, and eventually the ability to talk yourself. Many levels focus on playing around with the core mechanics in different ways.

I am overall happy with the game, but feel it has lost a lot of charm compared to Caption Quest, despite me having far longer to work on it. The focus on pushing core concepts to their limits makes for some quite confusing yet similar puzzles, and I feel it would have been more engaging to have simpler puzzles but with new, interesting mechanics quickly introduced, which the original game did well. Regardless, this was a good lesson in level design and had MUCH cleaner code than Caption Quest.

AutoRhythm

Made for my dissertation, AutoRhythm is a simple note-playing rhythm game in the style of DDR or Guitar Hero, in which the user can supply any song and have a level generated for it which they can then play.

The game itself was made in Unity, with accompanying Python code to deal with level generation. The user supplies an MP3 to the relevant folder, and a level can be generated, played, and saved for future use.

  • Options to control level generation (difficulty, style, etc.)
  • Easy rebinding with support for most HIDs and MIDI instruments
  • Customizable number of note streams (1-6 input buttons)
  • Level editor to tweak scores as needed

This project honestly worked better than expected, and can lead to some reasonably good levels for songs not featured in any rhythm games. It does not look very nice, but was intended to be more of a tool than an actual, polished rhythm game in it's own right. Endless options for improvements, but still a fun prototype.

Wizard Pool

A fun 2-player pool simulator made in Processing (a Java-style IDE + graphics library). Each turn, players can select a spell to cast, such as "moving pockets," "fog cloud," or "create portals."

A fun little project I occasionally update with new spells to enjoy with friends.
